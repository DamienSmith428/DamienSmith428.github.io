Common questions

Got questions?

What kind of software can I use NexAuth with?
Any software that can make HTTP requests. NexAuth exposes a standard REST API so you can integrate it with Python, JavaScript, C#, Java, C++, Rust, Go — literally any language. The ready-made client SDK is Python, but the API works with anything.
Do my users need to know about NexAuth?
No. Your users interact with your app's login screen, not NexAuth. The brand, the UI, the experience is entirely yours. NexAuth is invisible infrastructure — like how websites don't tell you which database they use.
Where is my users' data stored?
You get your own dedicated database on Render's infrastructure. Your users' data is completely isolated — it never mixes with any other NexAuth customer's data. You own the deployment.
How does HWID locking work?
On first login, the NexAuth client generates a hardware fingerprint from your user's machine (MAC address, CPU info, etc.) and registers it with your backend. On subsequent logins, the fingerprint is checked. If it doesn't match, login is rejected. You can reset any user's HWID from your admin panel.
Do I need to handle payments?
NexAuth doesn't handle payments — by design. You use whatever payment method fits your app: Stripe, PayPal, Ko-fi, even manual bank transfers. After payment you issue a redeem code or manually upgrade the user from your admin panel. This keeps NexAuth simple and you in control.
What happens if I exceed my plan's user limit?
New registrations will be rejected with a clear error until you upgrade your plan. Existing users are never affected. You'll see your user count in your admin panel so you can upgrade before you hit the limit.
Can I run multiple different apps on one plan?
Yes — Indie and above support multiple apps. Each app gets its own isolated user pool. Users of App A have no visibility into App B. The plan's end-user limit is shared across all your apps combined.
What's white-labeling?
On Pro and Studio plans, you can customize the login window, error messages, and branding so nothing says "NexAuth" anywhere your users can see. It looks like a system you built entirely in-house.
