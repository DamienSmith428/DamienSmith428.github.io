Everything you need to add production-grade licensing, authentication, and user management to your Python desktop app.

⚡ 5-minute Quick Start
NexAuth is designed to be dropped into any existing Python project. Most developers are up and running before they finish reading this page.

Quick Start

1. Deploy Your Backend

Click the button below to deploy your NexAuth backend to Render. This creates a Flask server + PostgreSQL database automatically.

Deploy to Render →

After deploying, copy your app URL (e.g. https://my-app.onrender.com). You'll need it next.

2. Add Files to Your Project

Download and add these two files to your project directory:

Terminal

Tiers & Licensing

NexAuth uses three built-in tiers. Every user gets exactly one tier at any time.

🎮 FREE

Default tier. Access to basic features. Perfect for free users and trial accounts.

💎 PREMIUM

Paying customers. Unlocks premium features. Also active during trial periods.

👑 ADMIN

Full access including admin panel. First registered user becomes admin automatically.

Tier Gating in Code

# Check tier in your app import auth # Simple boolean checks if auth.is_premium(): # True for premium AND admin unlock_downloader() if auth.is_admin(): show_admin_tab() # Or check the tier string directly if auth.current_tier == "premium": show_premium_badge() # Current user info print(auth.current_user) # "darkwolf92" print(auth.current_tier) # "premium" print(auth.current_email) # "user@email.com"

HWID Locking

Hardware ID locking binds a license to a specific machine, preventing account sharing. NexAuth generates a hardware fingerprint from MAC address, processor info, and machine name.

⚠️ HWID available on Indie plan and above
HWID locking requires the Indie ($9/mo) or Studio ($29/mo) plan.

How It Works

  1. On first login after HWID is enabled, the client generates and registers a hardware fingerprint
  2. On subsequent logins, the fingerprint is verified server-side
  3. If the fingerprint doesn't match, login is rejected with a clear error message
  4. Admins can reset any user's HWID from the user profile popup
  5. Users can self-service reset after a 7-day cooldown (configurable)

Trial Management

Trials give users temporary premium access. They expire automatically and the tier reverts to free without any action required.

# In your admin panel Python code auth.admin_set_trial( username = "newuser123", amount = 14, unit = "days" ) # In your app — check trial status if auth.trial_info: print(auth.trial_info) # "Trial active — 12h remaining"

API Reference — Authentication

POST /api/register Create a new user account
POST /api/login Authenticate and receive JWT token
GET /api/verify Verify token validity and get current tier
POST /api/redeem Redeem a tier upgrade code
POST /api/request_password_reset Request a password reset code from admin
POST /api/reset_password Complete password reset with admin-issued code

API Reference — Admin

GET /api/admin/users List all users with full details
POST /api/admin/set_tier Change a user's tier
POST /api/admin/set_trial Grant a time-limited trial to a user
POST /api/admin/ban_user Permanently ban a user with reason
POST /api/admin/suspend_user Temporarily suspend a user
POST /api/admin/unban_user Lift a ban or suspension
GET /api/admin/notifications Get unresolved admin notifications
POST /api/admin/user_ips Get last 3 login IPs for a user
DEL /api/admin/delete_user Permanently delete a user account
