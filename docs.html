NexAuth Docs

Everything you need to add user accounts, licensing tiers, and access control to your software. NexAuth works with any language or platform that can make HTTP requests.

⚡ 5-minute setup
Deploy your backend with one click, add two files to your project, set your server URL. Your app has real user accounts and tier-based access control before your coffee gets cold.

Overview

NexAuth is a hosted auth and licensing backend you deploy once and integrate into any software project. Here's the full picture of how it works:

  • You are a developer who wants user accounts and access control in your app
  • Your end-users are the people who use your software — they register and log in inside your app
  • NexAuth is the backend that handles everything between them: authentication, session tokens, tier checks, HWID verification, and admin controls
  • Your admin panel is embedded inside your own app — you manage your users without ever leaving your software

NexAuth never appears to your end-users. Your app shows your login screen, your branding, your experience. NexAuth is invisible infrastructure.

Quick Start

Step 1 — Deploy your backend

Click the Render deploy button from your NexAuth dashboard. This creates:

  • A dedicated Flask auth server running at https://your-app-name.onrender.com
  • A dedicated PostgreSQL database for your end-users' data
  • Automatic SSL, environment configuration, and database migrations

Copy your app URL — you'll need it in the next step.

Step 2 — Get the client files

Download the NexAuth client SDK for your language from your dashboard. The Python SDK comes as two files:

Tiers & Access Control

Every user in your app has exactly one tier. NexAuth provides three built-in tiers:

🎮 FREE

The default tier. New registrations start here. You decide what features free users can access.

💎 PREMIUM

Paying or trial users. is_premium() returns true. Unlocks whatever features you choose to gate.

👑 ADMIN

Full access. is_premium() also returns true for admin. The first registered user becomes admin automatically.

Gating Features

Check tier anywhere in your codebase. There's no SDK initialization or context object — just call the function.

# Boolean helpers — simplest approach if auth.is_premium(): # True for premium AND admin show_export_button() if auth.is_admin(): show_admin_tab() # Or check the tier string directly for more granular control tier = auth.current_tier # "free" | "premium" | "admin" if tier == "admin": show_crown_badge() elif tier == "premium": show_diamond_badge() else: show_free_badge()

HWID Locking

Hardware ID locking binds a license to a specific physical machine, preventing users from sharing account credentials with others.

⚠️ HWID requires Starter plan or above
HWID locking is available on the Starter ($5/mo) plan and all paid plans above it. The Free plan does not include HWID.

How It Works

  1. On the user's first login with HWID enabled, the client generates a hardware fingerprint (from MAC address, CPU identifier, machine name, etc.)
  2. The fingerprint is sent to your backend and stored against that user's account
  3. On every subsequent login, the fingerprint is re-generated and compared server-side
  4. If the fingerprints don't match, the login is rejected with a clear error message
  5. You can reset any user's HWID from the profile popup in your admin panel — useful when a user gets a new machine or reinstalls their OS

Security Notes

HWID fingerprints based on software-readable hardware identifiers (MAC address, CPU info) can be spoofed by determined users. For the vast majority of use cases — preventing casual account sharing — this is entirely effective. For high-security licensing (medical, financial, etc.), consider combining HWID with other measures.

Trials & Upgrades

Trials give users temporary premium access that expires automatically. You set the duration, NexAuth handles the timer and automatic reversion.

# Grant a 14-day trial from your admin panel code auth.admin_set_trial( username = "new_user", amount = 14, unit = "days" # minutes | hours | days | weeks ) # Check trial status in your app if auth.trial_info: show_trial_banner(auth.trial_info) # trial_info = "Trial active — 12 days, 4 hours remaining"

Connecting to Payment

NexAuth doesn't handle payments — by design. This keeps it simple and lets you use any payment method. Here are the two common patterns:

Redeem Codes (simplest): Generate a redeem code in your Render environment variables. After a user pays via Stripe, PayPal, Ko-fi, or any other method, you give them the code. They type it into your app's profile screen and their account upgrades automatically.

Manual Upgrade: User pays and contacts you. You open your admin panel inside your app, find their account, and click "Set Tier → Premium". Done in 10 seconds.

API Reference — Authentication

All endpoints are on your deployed backend URL. All protected endpoints require Authorization: Bearer <token>.

POST /api/register Register a new end-user account
POST /api/login Authenticate and receive a JWT token + tier
GET /api/verify Verify a token — returns current tier, username, trial info
POST /api/redeem Redeem a tier upgrade code
POST /api/request_password_reset User requests a password reset (creates admin notification)
POST /api/reset_password Complete reset with admin-issued code + new password
POST /api/profile/update Update own username or email (requires valid token)

API Reference — Admin

All admin endpoints require an admin-tier JWT token.

GET /api/admin/users List all end-users with full details
POST /api/admin/set_tier Change a user's tier (free / premium / admin)
POST /api/admin/set_trial Grant a time-limited trial to a user
POST /api/admin/set_email Set or update a user's email address
POST /api/admin/ban_user Permanently ban a user with reason
POST /api/admin/suspend_user Temporarily suspend a user (duration + reason)
POST /api/admin/unban_user Lift a ban or suspension
POST /api/admin/user_ips Get last 3 login IPs for a user
GET /api/admin/notifications Get unresolved admin notifications (password resets, flags)
POST /api/admin/resolve_notification Mark a notification as resolved
DEL /api/admin/delete_user Permanently delete a user account and all their data
